Hardware Deep Dive

What Each Component Actually Does

Inside the System

  • CPU executes instructions; cores handle parallel work.
  • RAM stores active data; more RAM helps multitasking.
  • Storage keeps data persistently (HDD/SSD/NVMe).
  • Motherboard connects buses, chipset, and peripherals.

Performance Clues

  • High disk usage + low RAM = paging/slowdowns.
  • Thermal throttling can reduce CPU performance.
  • Driver issues often present as unstable peripherals.
  • Power problems mimic random system instability.
Ports and Interfaces

Cables, Ports, and Practical Use

  • USB-A/USB-C: data transfer, charging, peripherals; know backward compatibility limits.
  • Display outputs: HDMI, DisplayPort; audio/video support and monitor compatibility.
  • Network: RJ45 Ethernet for stable low-latency connectivity.
  • Audio interfaces: analog jacks vs digital audio transport.
  • Know basic troubleshooting: wrong cable standard, damaged port, outdated driver, wrong input source.
Networking

Network Foundations You Need

Addressing + Name Resolution

  • IP identifies devices; DNS maps names to IPs.
  • Private vs public addressing concepts.
  • Default gateway reaches outside the subnet.

Common Devices

  • Modem: WAN handoff from ISP.
  • Router: pathing between networks.
  • Switch: local packet forwarding (Layer 2).
  • Access point: wireless bridge to wired LAN.
Network Triage

Fast Troubleshooting Path (No Internet)

  1. Check physical link: cable seated, Wi-Fi connected, adapter enabled.
  2. Confirm IP config: has valid local address and gateway.
  3. Test local gateway reachability first, then external IP.
  4. If IP works but names fail, suspect DNS issue.
  5. Reboot endpoint/network gear only after basic checks and documentation.

Exam traps: skipping scope checks and guessing root cause too early.

Mini Quiz 1

Core Concepts Check

  1. What is the key difference between bandwidth and throughput?
    Bandwidth is maximum capacity; throughput is the actual delivered rate after overhead/congestion.
  2. In one sentence, explain the difference between an incident and a problem.
    An incident is an immediate service disruption; a problem is the underlying cause of one or more incidents.
  3. Which cloud model gives customers the most infrastructure control: IaaS, PaaS, or SaaS?
    IaaS gives the most infrastructure-level control of the three.
  4. Why is identifying scope before troubleshooting considered critical?
    Scope tells you impact and likely fault domain, preventing random fixes and saving time.

Software Lifecycle

Database Essentials

Relational Thinking for Beginners

  • Tables store entities; rows are records; columns are attributes.
  • Primary key uniquely identifies each row.
  • Foreign key links related tables and enforces relationships.
  • Queries retrieve targeted data using filtering and sorting.
  • Indexes improve read speed but can impact write performance.
Mini Quiz 2

Networking and OS Check

  1. If a host can ping an external IP but cannot browse by domain, what service is most likely failing?
    DNS resolution is the likely failing service.
  2. What role does the default gateway play in network communication?
    It forwards traffic destined for networks outside the local subnet.
  3. Give one reason a machine with high disk usage and low RAM feels slow.
    Low RAM causes paging to disk, and disk I/O is much slower than memory access.
  4. What is one advantage of least privilege in endpoint administration?
    It limits blast radius by reducing what compromised or mistaken accounts can change.

Identity and Access

Authentication vs Authorization

Authentication

  • Confirms identity: password, token, biometric.
  • MFA combines two or more factors.
  • Account lockout/rate limits reduce brute-force risk.

Authorization

  • Defines what an authenticated user can do.
  • Role-based access simplifies management.
  • Periodic access reviews remove stale permissions.
Security Operations

Incident Response (Beginner Level)

  1. Identify and document suspicious activity quickly.
  2. Contain impacted systems to stop spread.
  3. Preserve evidence and escalate appropriately.
  4. Eradicate cause and recover services safely.
  5. Perform lessons-learned and improve controls.

Do not destroy evidence by making uncontrolled changes before documentation.

Mini Quiz 3

Security and Access Check

  1. What is the difference between authentication and authorization?
    Authentication verifies identity; authorization determines what actions/resources that identity can access.
  2. Name two warning signs that an email may be phishing.
    Urgent pressure language and mismatched/suspicious sender links are two common signs.
  3. What is the first priority after a likely account compromise: containment, documentation, or cleanup?
    Containment is first to stop further damage, then documentation and cleanup follow process.
  4. Why is MFA effective even when a password is exposed?
    Attackers still need an additional factor, so password-only theft is not enough for access.
Troubleshooting Method

Step-by-Step Process (Exam Priority)

Correct Sequence

  1. Identify problem.
  2. Establish theory.
  3. Test theory.
  4. Plan and implement fix.
  5. Verify full function.
  6. Document findings.

Common Mistakes

  • Jumping to fix without isolating scope
  • Changing multiple variables at once
  • Ignoring user communication during outage
  • Failing to record what changed
Scenario Drill 1

PC Is Slow + Browser Freezes

  • Gather symptoms: all apps or browser only? start time? recent changes?
  • Check resource usage: CPU, RAM, disk queue, startup programs.
  • Test likely causes: too many tabs, low memory, malware, storage bottleneck.
  • Apply fix: disable startup bloat, patch browser/OS, run security scan.
  • Validate by reproducing normal workload and documenting results.
Scenario Drill 2

User Clicked Suspicious Link

  • Ask what was clicked/downloaded and when it happened.
  • Disconnect affected device if compromise is likely.
  • Reset credentials from known-safe device.
  • Run endpoint scans and check unusual account activity.
  • Document, report, and provide user awareness follow-up.

Priority is containment first, then cleanup.

Scenario Drill 3

Cannot Print to Network Printer

Quick Checks

  • Printer power/ready state and paper/toner
  • Client on same network/VLAN path
  • Correct printer selected and set as default
  • Queue not stuck with failed job

Escalation Path

  • Restart spooler/service if needed
  • Update/reinstall print driver
  • Test from second endpoint for scope
  • Document final root cause + fix
Mini Quiz 4

Troubleshooting and Process Check

  1. Which troubleshooting step comes before implementing a fix?
    Test the theory to determine cause comes before planning/implementing the fix.
  2. Why should you change one variable at a time during diagnosis?
    It isolates causality so you can confidently attribute outcomes to a specific change.
  3. What should a strong ticket note always include after each action?
    Timestamp, action taken, observed result, and next step/decision.
  4. In exam questions, what clue words usually indicate process order matters most?
    Words like first, best initial, most likely next, and immediate indicate sequence priority.

Exam Tactics

How to Handle Tech+ Questions

Reading Strategy

  • Underline clue words: first, best, most secure, most likely.
  • Identify domain quickly: network, security, software, process.
  • Remove clearly wrong choices before deciding.

Time Strategy

  • Move fast on straightforward recall questions.
  • Flag scenario-heavy items and return later.
  • Leave no blanks; educated guesses beat empty answers.
